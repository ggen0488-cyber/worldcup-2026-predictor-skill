# 玄学模式说明 🔮（仅供娱乐）

可开关的趣味模块。默认关闭——用户说"开启玄学模式""算一卦""玄学预测"才启用。
四术合参，起卦以"对战双方代码 + 日期"为引子，**结果可复现**（同输入同卦象，用 SHA-256 派生确定性字节流，不依赖系统随机）。

## 四术

### ① 周易六十四卦
模拟"三枚铜钱"摇六爻（6 老阴变 / 7 少阳 / 8 少阴 / 9 老阳变），自下而上成卦。
得**本卦**（当前态势）+ **变卦**（发展走向）+ 变爻。卦辞与吉凶分（luck −2…+2）存于 `references/hexagrams.json`。有变爻时吉凶 = 本卦 0.6 + 变卦 0.4。

### ② 五行生克
按球队主色映射五行：红/橙→火，蓝/黑→水，白/金→金，绿→木，黄/棕→土。
- 相生：木→火→土→金→水→木（生人者泄气，被生者受益）
- 相克：木克土、土克水、水克火、火克金、金克木（克人者占上风）
- 同行比和，势均力敌。

### ③ 生肖纪年
2026 为**丙午马年**，火气当令：火队得气最旺(+0.4)，水次之(+0.2)，金被泄(−0.2)。

### ④ 数字命理
取球队 Elo 的数字根与日期数字根求"共振度"，越接近气场越合。

## 三种呈现方式（由 AI 助手按用户意图选择）
1. **纯玄学**：`divination.py A B` —— 只摇卦断吉凶，图个乐。
2. **科学+玄学并列**：先 `predict.py match`，再 `divination.py`，两份结果并排展示，用户自行品。
3. **玄学加权**：取 `divination.py A B --factor`（输出约 ±0.05 的气运修正因子，利甲方为正），由 AI 助手叠加到泊松胜率上得"开过光的概率"：
   `p_A' = clamp(p_A + factor, 0, 1)`，再与平/负归一化。叠加后务必注明"已掺入玄学，仅供娱乐"。

## 起卦背景快照

当用户要求当前比赛的玄学预测时，Agent 可以先收集比赛背景并保存 `data/live/divination_context.json`。脚本不联网，也不绑定固定来源；这些资料只用于确定起卦语境和展示，不证明玄学结论可靠。

可收集：

- 比赛日期、开球时间、举办城市、场地。
- 双方球队名称、颜色、国家/地区象征、方位或其他象意。
- 用户具体问题，例如“谁更有冠军相”“这场有没有冷门相”。
- 简要背景和来源 URL。

示例：

```json
{
  "_meta": {
    "collected_at": "2026-06-19T10:00:00Z",
    "sources": [
      {
        "name": "match center",
        "url": "https://example.com/match",
        "checked_at": "2026-06-19T10:00:00Z"
      }
    ]
  },
  "matches": [
    {
      "teams": ["ARG", "BRA"],
      "question": "这场谁的气势更顺？",
      "date": "2026-06-17",
      "divination_date": "0617",
      "kickoff_time": "2026-06-17T20:00:00-04:00",
      "city": "New York/New Jersey",
      "venue": "MetLife Stadium",
      "symbols": {
        "ARG": ["蓝白", "南方来客"],
        "BRA": ["黄绿", "桑巴"]
      },
      "notes": ["用于起卦背景展示，不作为可信预测证据"]
    }
  ]
}
```

规则：

- `--date` 参数优先级最高；未传时，脚本优先使用匹配对阵的 `divination_date`，再从比赛日期/开球时间提取 MMDD，最后回退到默认 `0617`。
- 伤停、预计首发、赔率、近期状态等理性情报应进入 `data/live/intelligence.json`；玄学上下文最多作为背景提及，不能说这些资料让玄学更可信。
- 如果来源不足，说明只能按默认日期或用户给定日期起卦，不要伪装成当前实时背景。

## 铁律
**任何玄学输出都必须附带：「⚠️ 玄学预测仅供娱乐，认真你就输了」**，不得包装成真实可信的预测。
