# 理性预测模型说明

三层模型：**Elo 评级 → Agent 情报修正 → 足球随机性修正泊松比分 → 蒙特卡洛模拟**。预测脚本本身不联网；如果用户要求“最新/当前”预测，由 Agent 先核验真实小组、赛程、赛果，并在单场预测前保存赛前情报快照。

## 1. 球队评级（Elo）
每队需要一个 `elo` 值（约 1500–2120）。脚本优先读取 `data/live/teams.json`，没有 live 快照时回退到 `references/teams.json` 样例数据。Agent 保存 live 快照时应保留 `elo`、`group`、`host`、`color` 等字段。

## 2. Agent 赛前情报修正

当存在 `data/live/intelligence.json` 时，脚本会读取 Agent 已保存的赛前情报快照：

- `teams.*.elo_delta`：团队级有界修正，限制在 `-80` 到 `+80` Elo 之间，影响单场、小组、夺冠和淘汰赛模拟。
- `matches[].goal_delta`：指定两队单场的进球期望修正，限制在 `-0.35` 到 `+0.35` 之间，并按 `confidence` 折算；当前只用于 `match A B` 命令。

这些修正来自伤停、预计首发、近期状态、赛程体能、场地天气、战术对位、比赛动机、赔率或评级变化等信息。情报只调整基础预期，不取消足球随机性，也不能包装成确定性结论。

## 3. 泊松比分模型
由两队 Elo 差换算"净胜球期望"(goal supremacy)，再拆成各自的期望进球 λ：

```
sup = (elo_A - elo_B) / 120          # 每 120 分约合 1 个净胜球
λ_A = 2.6/2 + sup/2 (+ 0.35 若A为东道主)
λ_B = 2.6/2 - sup/2 (+ 0.35 若B为东道主)
```

- `2.6` 为单场总进球的联赛均值；`0.35` 为东道主（美/加/墨）主场加成。
- 比分 (i, j) 概率 = Poisson(i; λ_A) × Poisson(j; λ_B)，对 0–8 球截断；单场输出会对若干状态冲击点做加权混合，而不是只使用一个静态 λ。
- 胜/平/负概率 = 对该矩阵按 i>j / i=j / i<j 求和。

### 主预测结果与比分分布

- 积分赛/小组赛的主预测赛果取胜/平/负三项中概率最高者。
- 主预测比分必须从与主预测赛果一致的比分里选择最高概率项。例如主方向是客胜，就不能把 `1-1` 包装成主预测比分，只能把它列为“单一最高概率比分/比分分布参考”。
- 淘汰赛的主预测结果是“谁晋级”，不是常规时间胜/平/负。简化公式为：

```
P(A晋级) = P(A常规时间胜) + P(平局) × P(A点球胜)
P(B晋级) = P(B常规时间胜) + P(平局) × P(B点球胜)
```

- 淘汰赛仍要给出常规时间比分；如果进入点球决胜，还要额外给出点球比分预测。
- `route TEAM` 会在整届模拟中统计该队每轮到达概率和常见潜在对手，用于回答“晋级路线/可能碰谁/走多远”。

## 4. 足球不稳定因素

足球单场样本小、进球少，冷门和偶然性显著。脚本用三层方式避免过度确定：

1. **泊松进球随机性**：即使期望进球固定，实际比分也会随机抽样。
2. **单场状态冲击**：未锁定的比赛在抽样前加入一次零均值冲击，模拟临场状态、战术匹配、伤停、红牌、天气、裁判尺度等未显式建模因素：

```
shock ~ Normal(0, σ)
λ_A' = max(0.15, λ_A + shock/2)
λ_B' = max(0.15, λ_B - shock/2)
```

3. **概率收缩**：单场胜平负输出会向中性足球基线 `(37%, 26%, 37%)` 轻微收缩，避免强队概率显得过满。

当前默认值内置在 `scripts/predict.py`：单场冲击 `σ=0.28`，胜平负概率收缩 `10%`。这些不是用户参数，而是模型默认假设，因为足球竞赛天然具有较高偶然性。

## 5. 蒙特卡洛整届模拟
默认 1 万次，每次：
1. **小组赛**：12 组各打 6 场单循环；积分→净胜球→进球数→随机排名。
2. **晋级**：各组前 2（24 队）+ 8 个最佳第三名 = 32 队；`group` 命令会整届同步模拟，因此单组晋级概率已包含最佳第三名路径。
3. **种子**：按小组赛战绩把 32 队从 1 到 32 排序。
4. **淘汰赛**：标准蛇形对阵（1v32…强弱分离），单场淘汰；平局按 Elo 轻微加权的点球决胜（scale 600，接近五五）。
5. 记录每队到达的最深轮次与冠军归属，多次聚合成概率。

## 6. 真实赛果锁定
`data/live/results.json` 与 `data/results.json` 中已结束的比赛会被**锁定**：小组赛用真实比分代替模拟；淘汰赛只在相同阶段复用真实比分。每条记录使用球队代码和阶段：

```json
{"home": "MEX", "away": "CRO", "hg": 1, "ag": 2, "stage": "GROUP"}
```

`stage` 可用 `GROUP`、`R32`、`R16`、`QF`、`SF`、`F`。这样同两队在不同阶段再次交手时，不会误用旧比分。

## 已知简化（可后续增强）
- 淘汰赛对阵采用"强弱分离蛇形种子"近似，非 FIFA 官方 R32 槽位映射表。
- 评级为静态值，未做赛中动态升降。
- 伤停、红牌、体能、天气等可通过 Agent 情报快照做有界修正；若没有情报快照，则只通过通用波动项近似，不做事件级建模。
