# HTML 预测报告

完成预测后，Agent 应生成一个本地 HTML 报告并自动打开。不要依赖额外脚本生成 HTML；由 Agent 根据本次预测输出、赛前情报和必要说明直接写出一个自包含 `.html` 文件。

优先使用 `predict.py --json` 和 `divination.py --json` 的结构化结果生成报告，避免从中文文本中反解析概率、比分和原因。

## 保存位置

- 默认保存到 `reports/`，文件名使用英文小写、数字和短横线，例如 `reports/arg-vs-bra-2026-prediction.html`。
- `reports/` 是运行产物，不应提交到仓库。
- HTML 必须自包含：CSS 和少量交互 JS 写在同一文件内，不依赖外部 CDN、字体、图片或构建工具。

## 内容结构

报告至少包含：

- 首屏球场：用 HTML/CSS/SVG 构建一个非品牌化足球场场景，包括草坪纹理、中线、禁区、球门、看台层次和灯光氛围。
- 真实比分牌风格展示：在球场上方或中央放置类似球场电子比分牌/转播记分牌的模块，显示双方队名、主预测比分、胜平负概率、预测时间和“预测”标签。比分牌必须突出概率最高的主预测结果，不得伪装成已完赛真实比分。
- 标题区：对阵双方、预测类型、生成时间、数据快照时间。
- 核心结论：概率最高的主预测赛果、主预测比分、胜/平/负概率、预期进球、冷门概率空间，或晋级/夺冠概率。
- 预测原因：用清晰短句说明为什么得到该主预测，至少覆盖评级/预期进球、赛前情报、赛制路径、随机性和关键风险中的相关项。
- 详细预测：比分分布 Top 5、双方预期进球、胜平负概率条、关键情报修正、模型默认随机性说明和置信度。单一最高概率比分若与主预测赛果不一致，只能标为“比分分布参考”。
- 观赛提醒：在结论区或页脚明确展示“预测结果仅供参考，请理性观赛。”。
- 模型信息：展示 `model_version`、数据快照时间和蒙特卡洛模拟误差；若脚本提供 `simulation_error_95_max`，用“约 ±X 个百分点”表达。
- 可视化：概率条、比分卡片、排名表、淘汰赛路径、冠军概率榜或赛程路径，按预测类型选择。
- 情报摘要：列出 Agent 收集的关键伤停、阵容、近期状态、赛程体能、战术对位和来源可信度。
- 场景推演：至少给出 2-3 条简短路径，例如“强队正常发挥”“低比分胶着”“冷门路径”，说明哪些条件可能改变结果。
- 不确定性说明：明确足球随机性、冷门空间和模型边界。
- 玄学区：仅在用户要求玄学时展示，并与理性预测分区；必须清楚展示本卦、变卦、变爻、卦辞/象意、五行生克、生肖纪年、数字命理、气运因子和断语原因。
- 来源区：列出本次使用的数据/情报来源 URL、采集时间和缺失说明。

## 视觉要求

- 做成可直接展示的球场大屏 + 数据仪表盘风格，而不是纯文本转储。
- 首屏必须有“球场”作为视觉主体：可以用 CSS 渐变、伪元素、内联 SVG 或 canvas 构建，但必须是自包含实现。
- 比分牌应有真实比赛现场感：深色面板、发光数字、队名分区、时间/状态条、概率小条和轻微阴影；必须清楚标注“预测比分”和“最高概率赛果”。
- 使用响应式布局，桌面和手机都能阅读；核心结论应在首屏可见。
- 使用高对比度、多色但克制的运动数据风格；不要使用 FIFA 官方 logo、赛事视觉、官方字体或受保护品牌资产。
- 概率条、卡片、标签和表格要能扫描，文本不能溢出或重叠。
- 可以使用 CSS 动画、微交互和 SVG/CSS 图形，但不要引入外部库。

## 详细信息清单

单场预测报告应尽量包含：

- 双方胜/平/负概率和概率条。
- 预测比分牌上的主预测比分，以及比分分布 Top 5。主预测比分必须与最高概率赛果方向一致。
- 双方预期进球、Elo 或评级差异、东道主/中立场说明。
- Agent 赛前情报：伤停、停赛、首发可能性、近期状态、休息天数、旅途和天气/场地影响。
- 战术对位：压迫、转换、定位球、防线速度、边路/中路优势。
- 预测原因说明：把模型输出和现实情报连接起来，例如“评级优势 + 预期进球更高 + 对方防线伤停 + 但单场随机性仍高”。
- 结果敏感点：哪些单一事件可能明显改变结果，例如早早进球、红牌、点球、关键球员缺阵。
- 数据新鲜度：快照时间、来源时间、缺失或冲突信息。

按赛制补充：

- 积分赛/小组赛/联赛：展示胜、平、负三项概率和主预测比分。
- 淘汰赛：展示晋级方、常规时间比分、加时/点球路径；若进入点球，额外展示点球比分预测。
- 出线预测：展示小组排名、晋级概率、前二概率、小组第一概率、最佳第三概率和关键赛果影响。
- 晋级路线预测：展示完整路线图、每轮潜在对手、每轮晋级概率、累计概率和最大风险节点。
- 夺冠预测：展示冠军概率梯队、进入决赛/四强/八强/32强概率，以及可能路径。

玄学预测报告必须包含：

- 起卦背景：对阵、用户问题、比赛日期或 `divination_date`、地点/方位/颜色象意、资料来源和缺失说明。
- 周易卦象：本卦代表当前态势，变卦代表走势，变爻代表转折点；如果无变爻，说明走势更偏稳定。
- 卦象解释：逐条解释卦辞/象意如何映射到双方气势、攻守节奏或冷门相，不得只写“某队更旺”。
- 五行、生肖、数字命理：分别说明颜色五行生克、2026 丙午马年气势、数字根共振如何影响娱乐判断。
- 气运因子：若使用 `--factor`，展示因子正负、利哪一方、如何叠加到理性概率；必须标明这是玄学维度，不要与理性模型依据混同。
- 玄学断语原因：用 3-5 条短句说明为什么给出这个玄学倾向，并列出相反象意或矛盾点。
- 边界说明：明确玄学不替代理性情报、赔率、伤停和模型概率。

## 自动打开

生成 HTML 后自动在默认浏览器打开。根据运行平台选择命令：

- Windows PowerShell: `Start-Process .\reports\file.html`
- macOS: `open reports/file.html`
- Linux: `xdg-open reports/file.html`

如果用户明确要求不要打开，只生成文件并返回路径。

## 输出给用户

最终答复应包含：

- HTML 文件路径。
- 是否已自动打开。
- 一句简短提醒：概率不是确定性结论；若包含玄学内容，说明它已与理性预测分区展示。
